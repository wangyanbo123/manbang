# 更新日志

> 简要记录仓库文档与 Demo 行为的重要变更，便于选手与评测对齐。

## 2026-05-08

- **复赛总时长上限（文档）**：《赛题详情》中复赛**总仿真运行时长**由「不超过 1 小时」调整为「**不超过 4 小时**」。见 `docs/01-赛题详情.md`。
- **参赛约束（文档补充）**：明确禁止决策代码**直接读取**货源 / 司机原始文件（如 `cargo_dataset.jsonl`、`drivers.json`）；货源与司机信息须通过仿真接口（如 `query_cargo`、`get_driver_status`、`query_decision_history` 等）获取。详见 `docs/02-数据说明.md`「1.1 信息获取约束」；`demo/README.md` 文首有简要重述。
- **司机信息与偏好**：更新 `demo/server/data/drivers.json`（及部分裁判侧同步数据）：调整多名司机的偏好条款（文案、罚金额度与封顶、窗口时间等），含**新增或改写**的规则。
- **偏好评测脚本**：`demo/calc_monthly_income.py` 已按上述司机配置重做各 `DriverDxxx` 偏好计算逻辑，罚分与明细字段与 JSON 中 `penalty_amount` / `penalty_cap` 对齐。
- **决策历史查询**：在 `SimulationApiPort` 中增加 `query_decision_history(driver_id, step)`。评测运行时，历史来自与仿真编排器**共享的内存列表**（与落盘 `actions_*.jsonl` 每条记录结构一致），`step=-1` 为本司机本会话全部步骤，正整数为末尾若干条（查看上一笔传 `1`）。
