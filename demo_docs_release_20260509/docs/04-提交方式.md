# 04 - 提交方式

## 1. 提交格式

仅接受 **ZIP 压缩包**，不接受仓库链接或其他格式。  
建议文件名：`队伍名_赛题提交.zip`。

## 2. 提交内容要求

选手提交 **ZIP 包内为 `demo/` 根目录**；**不必**提交与赛题包完全一致的 `server/`（评测使用赛方统一代码，无自行修改则无需打包），**不要**提交 `data/`（数据体积大，由赛方在评测环境提供，路径仍为 `server/data/`）。

**收益与本地脚本：** 正式评测将使用赛方统一流程 **重新计算** 收益与相关指标，**以评测端产出为最终依据**。赛题包中的 `calc_monthly_income.py` 仅便于选手 **本地自测与核对**，**不必**作为提交物；亦 **无需** 为通过提交来额外说明或附带「总收益」汇总文件——若本地曾生成 `monthly_income_*.json` 等，仅供自查，**不等于**官方成绩。

### 初赛

- 压缩包内 **至少包含** `demo/agent/` 与 `demo/results/`（与 `server` 平级）。
- `results/` 中应含本队仿真产生的 **可追溯产物**（如 `actions_*.jsonl`、`run_summary_*.json` 等，具体以当轮赛题说明为准）；其中本地收益 JSON **是否放入不影响官方重算**。
- 初赛最后一周将发布**新的数据集版本**，用于该阶段后续评测；请以赛方当期公告与数据包为准。
- 初赛阶段的**最终晋级标准**以该新版数据集上的官方评测得分为准。

### 复赛

- 压缩包内 **至少包含** `demo/agent/`。
- **不要**提交 `demo/results/`（无需附带仿真结果目录）。
- 复赛将使用**新的数据集版本**（不同于初赛公开数据）；该数据集**不对外公开发布**，由主办方在本地评测环境运行选手提交代码并计算成绩。

## 3. 推荐目录结构（提交包示例）

```text
demo/
├─ agent/
└─ results/               # 仅初赛须打包；复赛勿包含
```

**补充说明：** 若在最小目录之外 **另有须一并提交的文件或目录**（例如对 `simkit/` 的修改、辅助脚本、配置片段等），须 **同时附带说明文档**（建议置于 `demo/` 内，如 `SUBMISSION.md` 或增补 `README.md`），写清文件清单、用途、与官方赛题包的差异及评测侧需如何加载；无说明的附加内容可能无法纳入评测流程。

## 4. 提交前自检清单

- `demo/` 根目录名与层级规范；**勿**将大体积 `data/` 打入压缩包
- 初赛：已包含 `agent/` 与 `results/`，且 `results/` 内文件完整可读
- 复赛：已包含 `agent/`
- 配置文件、密钥等敏感信息勿写入仓库或提交包
- 本地可按《05-快速开始》自测；评测侧将使用统一 `server`（及数据）运行 `agent/`
