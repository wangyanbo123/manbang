# 05 - 快速开始

本章提供从安装到出结果的最短路径，按顺序执行即可完成一次本地评测。

## 1. 环境准备

在 `demo/server` 安装依赖：

```bash
cd demo/server
pip install -r requirements.txt
```

初始化配置文件（Windows）：

```bash
copy config\config.example.json config\config.json
```

**模型与密钥：** 在 `server/config/config.json`（由 `config.example.json` 复制）中可填写 `model_api_url`、`model_name`、`model_api_key` 等；亦可通过环境变量注入密钥（优先级高于配置文件）：`DASHSCOPE_API_KEY` 或 `TIANCHI_MODEL_API_KEY`，避免将真实密钥写入文件。若 **改用其他模型或调用方式**（例如替换网关、多模型路由、非 OpenAI 兼容接口），除在代码中实现外，须按《04-提交方式》**随包提交说明文档**（路径、鉴权、与默认 `ModelGatewayClient` 的差异等），否则评测环境可能无法复现。

## 2. 运行仿真

在 `demo/server` 执行：

```bash
python main.py
```

如需指定配置文件：

```bash
python main.py config/config.json
```

成功后会生成：

- `demo/results/actions_202603_*.jsonl`
- `demo/results/run_summary_202603.json`

## 3. 计算收益与校验

在 `demo` 目录执行：

```bash
cd ..
python calc_monthly_income.py
```

生成文件：

- `demo/results/monthly_income_202603.json`

## 4. 如何判断运行正常

请至少检查以下三点：

- `actions_*.jsonl` 非空，且每步包含时间与位置字段
- `run_summary_202603.json` 中有本次仿真关键参数与统计
- `monthly_income_202603.json` 可正常解析，`summary` 字段完整

## 5. 决策中如何查看历史步骤（可选）

评测 Demo 在进程内提供 `SimulationApiPort.query_decision_history(driver_id, step)`，可在一步决策中读取**本会话已累计**的动作记录（与最终 `actions_*.jsonl` 对齐）。参数 `step`：`-1` 为全部，正整数为末尾若干条（若只需上一笔则传 `1`）。详见 `demo/README.md`「决策环境接口」。

默认示例：`agent/model_decision_service.py` 每步仅打日志示例，不把历史写入模型上下文。

**禁止**：决策代码不得直接打开或解析 `cargo_dataset.jsonl`、`drivers.json` 等原始数据路径；须仅通过接口访问（见 `02-数据说明.md`「1.1」）。

## 6. 常见问题

- 收益为 0  
  查看 `monthly_income_202603.json` 中对应司机的 `validation_error`。  
  校验失败按司机隔离，不会导致全体司机清零。

- 偏好罚分不理解  
  查看 `drivers[*].preference_check.rules`，每条规则会给出中文说明、偏好原文、命中次数与罚分。

- 时间口径疑问  
  系统统一使用分钟（`simulation_progress_minutes`）；  
  天数相关窗口以 `run_summary_202603.json` 的 `simulation_duration_days` 为准。
