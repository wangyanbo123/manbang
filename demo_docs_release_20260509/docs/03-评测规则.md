# 03 - 评测规则

本章说明评测执行顺序、合法性校验口径和收益计算方式。

## 1. 评测执行顺序

1. 运行仿真：`demo/server/main.py`
2. 生成动作日志：`demo/results/actions_202603_*.jsonl`
3. 运行收益脚本：`demo/calc_monthly_income.py`
4. 生成评分结果：`demo/results/monthly_income_202603.json`

评测以最终结果文件为准。

## 2. 动作合法性校验

收益脚本会逐步校验每位司机的动作记录，重点包括：

- 时间一致性  
  - `step_elapsed_minutes = query_scan_cost_minutes + action_exec_cost_minutes`
  - 本步结束时间与上一步结束时间之差，必须等于本步总耗时

- 位置一致性  
  - `wait`：前后位置不得变化  
  - `reposition`：结束位置必须等于目标位置  
  - `take_order`（成功）：结束位置必须等于货源终点

- 耗时一致性  
  - `reposition`：按距离和速度计算并向上取整到分钟  
  - `take_order`（成功）：空驶耗时 + 装货等待耗时 + 干线耗时

- 参数与结果一致性  
  - 动作参数、结果详情、原始货源数据必须相互匹配

校验失败按司机隔离处理：单个司机失败不会影响其他司机计算。

## 3. 收益计算公式

对每位司机：

- `gross_income`：成功接单的货源价格总和（按 `cargo_id` 回查原始数据）
- `distance_km`：空驶里程 + 成功接单干线里程
- `cost = distance_km * cost_per_km`
- `net_income = gross_income - cost - preference_penalty`

## 4. 偏好罚分规则

有偏好的司机会执行额外校验。以下金额为当前示例口径（均为扣钱），用于说明评测机制；初赛二版数据与复赛数据会采用不同的偏好规则与扣钱参数。

- D006（每天至少连续休息 5 小时）：每天不满足扣 `400`，累计上限 `4000`
- D007（23:00-04:00 不接单不空驶）：每天不满足扣 `500`，累计上限 `5000`
- D008（每月至少 2 天完全不出车）：整月不满足一次性扣 `3000`
- D009（每天 23:00 前回家且到次日 08:00 不再接单或空驶）：每天不满足扣 `600`，累计上限 `6000`
- D010 规则1（每月至少 5 天到达目标点）：整月不满足一次性扣 `3000`
- D010 规则2（禁止进入禁入区域）：进入即一次性扣 `3000`

罚分体现在 `monthly_income_202603.json` 中每位司机的 `preference_penalty`、`preference_check`，并计入 `summary` 汇总。

偏好规则判定结果会输出在结果文件中，包含中文规则说明和偏好原文。

## 5. 结果文件解读

`monthly_income_202603.json` 主要字段：

- `drivers[*].income`：该司机收益明细
- `drivers[*].preference_check.rules`：偏好逐条判定
- `drivers[*].calculation_aborted` / `drivers[*].validation_error`：是否校验失败及原因
- `summary`：全体汇总（净收益、总罚分、token 统计、失败司机列表）
